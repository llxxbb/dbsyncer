## 背景

同步任务在读取源表或写入目标表时，可能遇到暂时性环境问题（死锁、超时、连接中断等）。当前重试逻辑硬编码在多处，关键字、重试次数、等待间隔全部写死。

## 决策

**用统一重试组件替代现有硬编码重试。**

改造方式：
1. 新增 `RetryInterceptor` + `KeywordMatcher` + `RetryPolicy`
2. `SqlServerCTListener` 主循环的重试逻辑 → 替换为 `retryInterceptor.execute()` 调用
3. `SqlServerCTQueryUtil.queryWithReadUncommitted()` 内部重试 → 替换为 `retryInterceptor.execute()` 调用
4. 删除 `isRetryableError()`、`MAX_RETRY_PER_VERSION`、`currentVersionRetryCount`
5. 配置：全局默认 + 任务级覆盖

详细设计见 [实施说明](../history/0013/impl-01-notes.md)。

## 影响

- 消除硬编码重试逻辑，统一为一个组件
- 重试关键字、次数、间隔全部可配
- 不影响正常路径性能

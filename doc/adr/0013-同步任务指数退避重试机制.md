## 背景

同步任务在读取源表或写入目标表时，可能遇到暂时性环境问题（死锁、超时、连接中断等）。当前重试逻辑硬编码在多处，关键字、重试次数、等待间隔全部写死。

## 决策

**用统一重试组件替代现有硬编码重试。**

改造方式：
1. 新增 `RetryInterceptor` + `KeywordMatcher` + `RetryPolicy` + `RetryConfig` + `SpringContextUtil`
2. `SqlServerCTListener` 主循环的重试逻辑 → 替换为 `retryInterceptor.execute()` 调用（源端读取重试）
3. `SqlServerCTQueryUtil.queryWithReadUncommitted()` 内部重试 → **已移除**（与外层 pull 重试嵌套，保留一层即可）
4. `AbstractDatabaseConnector.executeWriter()` 增加 `retryInterceptor.execute()`（目标端写入重试，所有数据库连接器统一覆盖）
5. 删除 `isRetryableError()`、`MAX_RETRY_PER_VERSION`、`currentVersionRetryCount`
6. 配置：`RetryConfig` 全局默认（`dbsyncer.retry.global.*`），连接器直接复用同一份全局策略
7. MySQL 源端无需改造——`BinaryLogRemoteClient` 底层已处理网络重连

详细设计见 [实施说明](../history/0013/impl-01-notes.md)、[设计变更](../history/0013/design-changes.md)。

## 影响

- 消除硬编码重试逻辑，统一为一个组件
- 重试关键字、次数、间隔全部可配
- 不影响正常路径性能

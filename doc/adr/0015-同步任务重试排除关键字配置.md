## 背景

ADR-0013 引入了统一的重试机制，允许通过配置关键字来决定哪些异常需要重试。然而，在某些场景下，某些异常是**确定性的、不可恢复的**（例如：字段类型不匹配、权限不足、表不存在等），对这些异常进行重试只会浪费时间，甚至可能导致不必要的通知风暴。

当前问题：
- 所有匹配重试关键字的异常都会被重试，无法区分"可恢复"和"不可恢复"异常
- 对于配置错误、权限问题等一次性错误，重试没有意义
- 用户无法通过配置快速跳过某些已知不可恢复的异常类型

## 决策

**在 RetryPolicy 中增加 `excludeKeywords` 配置项，与 `keywords` 并列。当异常消息命中 `excludeKeywords` 时，立即抛出异常，不执行重试；只有未命中排除关键字时，才检查 `keywords` 决定是否重试。**

### 架构设计

#### 1. RetryPolicy 结构

```java
public class RetryPolicy {
    // 重试关键字列表（命中后重试）
    private List<String> keywords;
    
    // 排除关键字列表（命中后直接抛出，不重试）
    private List<String> excludeKeywords;
    
    // 关键字匹配模式
    private MatchMode matchMode = MatchMode.CONTAINS_IGNORE_CASE;
    
    // 缓存的关键字匹配器
    private volatile KeywordMatcher matcher;
    
    // 缓存的排除关键字匹配器
    private volatile KeywordMatcher excludeMatcher;
}
```

#### 2. 配置绑定类 RetryConfig

```yaml
# 全局默认配置
dbsyncer.retry.global:
  use-keyword: true
  keywords: "deadlock,timeout,connection reset"
  exclude-keywords: "syntax error,permission denied,column not found"
  
# 任务级覆盖（优先级高于全局）
dbsyncer.retry.task.1001:
  use-keyword: true
  keywords: "deadlock,timeout"
  exclude-keywords: "syntax error,permission denied"
```

### 3. 执行流程（RetryInterceptor）

**架构设计：正常路径零开销**

```
┌─────────────────────────────────────────────────────────┐
│              RetryInterceptor.execute()                 │
└─────────────────────────────────────────────────────────┘
                          │
        ┌─────────────────┴─────────────────┐

        │                                   │
        ▼                                   ▼
┌──────────────┐                  ┌──────────────┐

                                    │ 抛出异常  │  │ 记录日志 │

                                    └──────────┘  └──────────┘
```

**核心原则**：
1. **正常路径零开销**：正常执行 `operation.get()`，无任何条件检查
2. **异常路径决策**：仅在捕获异常后才进入重试决策流程
3. **条件后置**：终止条件检查仅在异常发生时才执行
4. **避免污染**：不污染正常路径的性能和可读性

### 匹配规则

1. **匹配顺序**：先检查 `excludeKeywords`，再检查 `keywords`
2. **优先级**：排除关键字优先级高于重试关键字
3. **匹配模式**：
   - 使用 `KeywordMatcher` 进行匹配
   - 默认采用 `CONTAINS_IGNORE_CASE`（大小写不敏感包含匹配）
   - 支持 `MatchMode` 枚举：`EXACT`, `CONTAINS`, `CASE_INSENSITIVE`, `CONTAINS_IGNORE_CASE`
4. **匹配逻辑**：
   - `excludeKeywords` 命中 → `isExcludeKeywordMatch()` 返回 true → 不重试
   - `excludeKeywords` 未命中 + `keywords` 命中 → 重试
   - `excludeKeywords` 未命中 + `keywords` 未命中 + `useKeyword=false` → 重试

### 实现细节

#### RetryPolicy.java

```java
public class RetryPolicy {
    private List<String> keywords;
    private List<String> excludeKeywords;
    private volatile KeywordMatcher matcher;
    private volatile KeywordMatcher excludeMatcher;
    
    // Getters and Setters
    public List<String> getKeywords() { ... }
    public void setKeywords(List<String> keywords) {
        this.keywords = keywords;
        this.matcher = null;
    }
    
    public List<String> getExcludeKeywords() { ... }
    public void setExcludeKeywords(List<String> excludeKeywords) {
        this.excludeKeywords = excludeKeywords;
        this.excludeMatcher = null;
    }
    
    public void setMatchMode(MatchMode matchMode) {
        this.matchMode = matchMode;
        this.matcher = null;
        this.excludeMatcher = null;
    }
    
    // 判断是否匹配排除关键字
    public boolean isExcludeKeywordMatch(String message) {
        if (message == null || excludeKeywords == null || excludeKeywords.isEmpty()) {
            return false;
        }
        return getExcludeMatcher().matches(message);
    }
    
    // 判断是否匹配重试关键字
    public boolean isKeywordMatch(String message) {
        if (!useKeyword) {
            return true;
        }
        if (message == null || keywords == null || keywords.isEmpty()) {
            return false;
        }
        return getMatcher().matches(message);
    }
    
    // 懒加载排除关键字匹配器
    KeywordMatcher getExcludeMatcher() {
        if (excludeMatcher == null) {
            excludeMatcher = new KeywordMatcher(excludeKeywords, matchMode);
        }
        return excludeMatcher;
    }
}
```

#### RetryInterceptor.java

```java
public class RetryInterceptor {
    
    /**
     * 判断异常是否应该重试
     */
    private boolean shouldRetry(Exception e, RetryPolicy policy, String message) {
        // 先检查排除关键字（命中则立即抛出，不重试）
        if (policy.isExcludeKeywordMatch(message)) {
            return false;
        }
        // 未命中排除关键字后，再检查重试关键字
        if (!policy.isUseKeyword()) {
            return true;
        }
        return policy.isKeywordMatch(message);
    }
    
    /**
     * 查找实际匹配的关键字（用于日志）
     */
    private String findMatchedKeyword(String message, RetryPolicy policy) {
        if (message == null) {
            return null;
        }
        
        // 优先检查排除关键字
        List<String> excludeKeywords = policy.getExcludeKeywords();
        if (excludeKeywords != null && !excludeKeywords.isEmpty()) {
            KeywordMatcher excludeMatcher = policy.getExcludeMatcher();
            for (String keyword : excludeKeywords) {
                if (keyword != null && excludeMatcher.match(message, keyword)) {
                    return "EXCLUDED:" + keyword;
                }
            }
        }
        
        // 再检查重试关键字
        List<String> keywords = policy.getKeywords();
        if (keywords == null || keywords.isEmpty()) {
            return null;
        }
        KeywordMatcher matcher = policy.getMatcher();
        for (String keyword : keywords) {
            if (keyword != null && matcher.match(message, keyword)) {
                return keyword;
            }
        }
        return null;
    }
}
```

## 影响

### 优点

1. **减少无效重试**：对确定性错误立即失败，避免浪费时间
2. **降低通知频率**：避免因重试导致的重复通知
3. **提高可配置性**：用户可根据实际业务场景自定义排除规则
4. **向后兼容**：默认不启用（`excludeKeywords` 为空），不影响现有行为
5. **优先级清晰**：排除关键字优先于重试关键字，逻辑明确
6. **正常路径零开销**：99.9% 的正常调用不执行任何条件检查，性能最优
7. **异常路径明确**：所有重试决策逻辑集中在异常处理块中，代码清晰

### 缺点

1. **配置复杂度增加**：需要维护两套关键字（重试 + 排除）
2. **匹配冲突处理**：如果某异常同时匹配两套关键字，以排除优先（已明确设计）
3. **内存占用微增**：需缓存两个 `KeywordMatcher` 实例

## 配置示例

### 完整示例

```yaml
# 全局重试配置
dbsyncer.retry.global:
  # 是否禁用重试
  disable: false
  
  # 重试参数
  max-attempts: 5
  initial-interval: 1000
  max-interval: 60000
  multiplier: 2.0
  max-duration: 3600000
  
  # 关键字匹配模式
  use-keyword: true
  match-mode: CONTAINS_IGNORE_CASE
  
  # 重试关键字（命中后重试）
  keywords: "deadlock,timeout,connection reset,temporary failure"
  
  # 排除关键字（命中后直接抛出，不重试）
  exclude-keywords: "syntax error,permission denied,column not found,table does not exist,type mismatch"
```

### 任务级覆盖示例

```yaml
# 任务 1001：允许更多重试，但排除特定错误
dbsyncer.retry.task.1001:
  max-attempts: 10
  use-keyword: true
  keywords: "deadlock,timeout"
  exclude-keywords: "syntax error,permission denied"

# 任务 2002：完全禁用重试
dbsyncer.retry.task.2002:
  disable: true
```

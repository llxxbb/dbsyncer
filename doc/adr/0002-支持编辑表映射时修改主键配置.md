# ADR 0002: 支持编辑表映射时修改主键配置

**日期**: 2026-03-16  
**状态**: Accepted  
**作者**: 凌曦  
**修正日期**: 2026-03-17（根据审查结果修正）

---

## 背景与问题

当前在字段映射编辑界面可以指定目标表主键：

```html
<!-- 目标表主键输入 -->
<input id="targetTablePK" class="form-control" type="text" data-role="tagsinput"/>
```

**现状**：
- ✅ **添加时**：主键设置会生效。前端传递 `targetTablePK` 参数（逗号分隔），后端 `TableGroupChecker.getTable()` 方法解析并设置 `field.setPk(true)`。
- ❌ **编辑时**：主键修改**不会生效**，且配置**无法持久化**。

**影响**：
1. 用户编辑表映射时，如果发现主键配置错误，无法修正
2. 只能删除重建，用户体验差
3. 可能导致数据同步异常（主键配置错误）

---

## 核心问题（审查发现）

### 问题 1: targetTablePK 没有持久化 ❌

**根本原因**：
- `targetTablePK` 参数作为临时请求参数传递，**没有存储到 TableGroup 对象中**
- 编辑时如果不传递该参数，后端无法获取用户之前配置的主键顺序

**解决方案**：
在 `TableGroup` 类中新增 `targetTablePK` 字段用于持久化：

```java
public class TableGroup extends AbstractConfigModel {
    // 新增字段
    private String targetTablePK;  // 目标表主键配置（逗号分隔，按顺序）
    
    // Getter 和 Setter
    public String getTargetTablePK() { return targetTablePK; }
    public void setTargetTablePK(String targetTablePK) { this.targetTablePK = targetTablePK; }
}
```

**持久化流程**：
```
前端传递 → tableGroup.setTargetTablePK("ID,UserName,Age") 
         → JsonUtil.objToJson(tableGroup) 
         → Lucene Document (/data/config)
         → 反序列化后 tableGroup.getTargetTablePK() = "ID,UserName,Age"
```

---

### 问题 2: DDL 执行方式与项目框架不一致 ❌

**原方案问题**：
```java
// 直接使用 executeDDL()，绕过项目 DDL 框架
executeDDL(targetConnectorInstance, alterPkSql);
```

**项目现有 DDL 模式**：
```java
// 使用 connectorFactory.writerDDL() 框架
DDLConfig ddlConfig = new DDLConfig();
ddlConfig.setSql("/*dbs*/" + sql);  // /*dbs*/ 前缀防止双向同步循环

Result result = connectorFactory.writerDDL(
    connectorFactory.connect(targetConnector.getConfig()),
    ddlConfig,
    null);

if (StringUtil.isNotBlank(result.error)) {
    throw new RuntimeException("执行 DDL 失败：" + result.error);
}
```

**修正方案**：使用项目现有的 `connectorFactory.writerDDL()` 框架。

---

### 问题 3: 数据库兼容性不足 ❌

**原方案**：只考虑 MySQL 语法

**修正方案**：同时支持 MySQL 和 SQL Server

| 数据库 | DROP PRIMARY KEY | ADD PRIMARY KEY |
|--------|-----------------|-----------------|
| **MySQL** | `DROP PRIMARY KEY` | `ADD PRIMARY KEY (col1, col2)` |
| **SQL Server** | `DROP CONSTRAINT [PK_name]` | `ADD PRIMARY KEY (col1, col2)` |

---

## 决策

**支持在编辑表映射时修改主键配置，并执行 DDL 更新目标表约束**。

**改造范围**：
1. ✅ 后端：`TableGroup` 新增 `targetTablePK` 字段用于持久化
2. ✅ 后端：`TableGroupChecker.checkEditConfigModel()` 支持读取并应用主键参数
3. ✅ 后端：`TableGroupServiceImpl.alterPrimaryKey()` 使用项目 DDL 框架，支持 MySQL + SQL Server
4. ✅ 后端：调整事务顺序（先 DDL，后保存配置）
5. ✅ 前端：新增"主键顺序"按钮和对话框（带风险提示）
6. ✅ 前端：鼠标悬停按钮时显示当前主键顺序（tooltip）

```java
private Table getTable(String connectorId, String tableName, String primaryKeyStr) throws Exception {
    MetaInfo metaInfo = parserComponent.getMetaInfo(connectorId, tableName);
    Assert.notNull(metaInfo, "无法获取连接器表信息:" + tableName);
    
    // 自定义主键
    if (StringUtil.isNotBlank(primaryKeyStr) && !CollectionUtils.isEmpty(metaInfo.getColumn())) {
        String[] pks = StringUtil.split(primaryKeyStr, StringUtil.COMMA);
        Arrays.stream(pks).forEach(pk -> {
            for (Field field : metaInfo.getColumn()) {
                if (StringUtil.equalsIgnoreCase(field.getName(), pk)) {
                    field.setPk(true);
                    break;
                }
            }
        });
    }
    return new Table(tableName, metaInfo.getTableType(), metaInfo.getColumn(), metaInfo.getSql(), metaInfo.getIndexType());
}
```

### 3. 后端改造 - DDL 执行方法（使用项目框架）

**文件**: `dbsyncer-biz/src/main/java/org/dbsyncer/biz/impl/TableGroupServiceImpl.java`

**修正后的方法**:
```java
/**
 * 修改目标表的主键约束
 * 使用项目的 DDL 处理框架，支持 MySQL 和 SQL Server
 * 
 * @param tableGroup 表组对象
 * @param targetConnector 目标连接器
 * @param oldPrimaryKeys 原主键列表
 * @param newPrimaryKeys 新主键列表
 * @throws Exception 执行异常
 */
public void alterPrimaryKey(TableGroup tableGroup, Connector targetConnector, 
                            List<String> oldPrimaryKeys, List<String> newPrimaryKeys) throws Exception {
    if (CollectionUtils.isEmpty(newPrimaryKeys)) {
        throw new RuntimeException("新主键列表不能为空");
    }
    
    // 如果主键没有变化，跳过
    if (oldPrimaryKeys.equals(newPrimaryKeys)) {
        logger.info("主键未变化，跳过 DDL 执行");
        return;
    }
    
    // 获取数据库类型
    String connectorType = targetConnector.getConfig().getConnectorType();
    
    // 获取 SqlTemplate 构建 DDL
    AbstractDatabaseConnector dbConnector = 
        (AbstractDatabaseConnector) connectorFactory.getConnectorService(connectorType);
    SqlTemplate sqlTemplate = dbConnector.sqlTemplate;
    
    String tableName = tableGroup.getTargetTable().getName();
    String schema = ((DatabaseConfig) targetConnector.getConfig()).getSchema();
    
    // 构建 DDL（根据数据库类型）
    String alterPkSql;
    if (ConnectorType.MYSQL.equals(connectorType)) {
        alterPkSql = buildMySQLAlterPrimaryKeySql(sqlTemplate, schema, tableName, oldPrimaryKeys, newPrimaryKeys);
    } else if (ConnectorType.SQLSERVER.equals(connectorType)) {
        alterPkSql = buildSQLServerAlterPrimaryKeySql(sqlTemplate, schema, tableName, oldPrimaryKeys, newPrimaryKeys);
    } else {
        throw new UnsupportedOperationException("暂不支持该数据库类型的主键修改：" + connectorType);
    }
    
    // 使用项目的 DDL 框架执行
    DDLConfig ddlConfig = new DDLConfig();
    ddlConfig.setSql("/*dbs*/" + alterPkSql);  // 添加前缀防止双向同步循环
    
    Result result = connectorFactory.writerDDL(
        connectorFactory.connect(targetConnector.getConfig()),
        ddlConfig,
        null);
    
    if (StringUtil.isNotBlank(result.error)) {
        throw new RuntimeException("执行 DDL 失败：" + result.error);
    }
    
    logger.info("修改主键约束成功：{} -> {}", oldPrimaryKeys, newPrimaryKeys);
}

/**
 * MySQL 语法：ALTER TABLE xxx DROP PRIMARY KEY, ADD PRIMARY KEY (...)
 */
private String buildMySQLAlterPrimaryKeySql(SqlTemplate sqlTemplate, String schema, String tableName,
                                            List<String> oldPrimaryKeys, List<String> newPrimaryKeys) {
    String quotedTableName = sqlTemplate.buildTable(schema, tableName);
    String quotedKeys = newPrimaryKeys.stream()
            .map(pk -> sqlTemplate.buildColumn(pk))
            .collect(Collectors.joining(", "));
    
    if (!CollectionUtils.isEmpty(oldPrimaryKeys)) {
        return "ALTER TABLE " + quotedTableName + " DROP PRIMARY KEY, ADD PRIMARY KEY (" + quotedKeys + ")";
    } else {
        return "ALTER TABLE " + quotedTableName + " ADD PRIMARY KEY (" + quotedKeys + ")";
    }
}

/**
 * SQL Server 语法：需要先查询主键约束名
 */
private String buildSQLServerAlterPrimaryKeySql(SqlTemplate sqlTemplate, String schema, String tableName,
                                                 List<String> oldPrimaryKeys, List<String> newPrimaryKeys) throws Exception {
    String quotedTableName = sqlTemplate.buildTable(schema, tableName);
    String quotedKeys = newPrimaryKeys.stream()
            .map(pk -> sqlTemplate.buildColumn(pk))
            .collect(Collectors.joining(", "));
    
    StringBuilder sql = new StringBuilder();
    
    // 如果有原主键，先 DROP CONSTRAINT
    if (!CollectionUtils.isEmpty(oldPrimaryKeys)) {
        // SQL Server 需要查询主键约束名
        String pkConstraintName = getSQLServerPrimaryKeyConstraintName(schema, tableName);
        if (StringUtil.isNotBlank(pkConstraintName)) {
            sql.append("ALTER TABLE ").append(quotedTableName)
               .append(" DROP CONSTRAINT ").append(pkConstraintName)
               .append(", ADD PRIMARY KEY (").append(quotedKeys).append(")");
        } else {
            sql.append("ALTER TABLE ").append(quotedTableName)
               .append(" ADD PRIMARY KEY (").append(quotedKeys).append(")");
        }
    } else {
        sql.append("ALTER TABLE ").append(quotedTableName)
           .append(" ADD PRIMARY KEY (").append(quotedKeys).append(")");
    }
    
    return sql.toString();
}

/**
 * SQL Server：查询主键约束名（简化实现）
 * 后续可优化为查询系统视图 sys.key_constraints
 */
private String getSQLServerPrimaryKeyConstraintName(String schema, String tableName) {
    // 简化实现：返回 PK_表名 作为默认约束名
    return "PK_" + tableName;
}
```

### 4. 后端改造 - 调整事务顺序

**文件**: `dbsyncer-biz/src/main/java/org/dbsyncer/biz/impl/TableGroupServiceImpl.java`

**修改位置**: `edit()` 方法

**修正后的代码**（先 DDL，后保存）:
```java
@Override
public String edit(Map<String, String> params) throws Exception {
    String id = params.get(ConfigConstant.CONFIG_MODEL_ID);
    TableGroup tableGroup = profileComponent.getTableGroup(id);
    Assert.notNull(tableGroup, "Can not find tableGroup.");
    Mapping mapping = profileComponent.getMapping(tableGroup.getMappingId());
    mapping.assertDisableEdit();

    // 保存原主键信息（在 checkEditConfigModel 修改之前）
    List<String> oldPrimaryKeys = PrimaryKeyUtil.findTablePrimaryKeys(tableGroup.getTargetTable());
    
    // 解析新主键参数
    String targetTablePK = params.get("targetTablePK");
    List<String> newPrimaryKeys = new ArrayList<>();
    if (StringUtil.isNotBlank(targetTablePK)) {
        String[] pks = StringUtil.split(targetTablePK, StringUtil.COMMA);
        for (String pk : pks) {
            newPrimaryKeys.add(pk.trim());
        }
    }

    // 应用主键参数（内存修改）
    TableGroup model = (TableGroup) tableGroupChecker.checkEditConfigModel(params);
    
    // 【修正】先执行 DDL，成功后再保存配置
    if (!oldPrimaryKeys.equals(newPrimaryKeys)) {
        Connector targetConnector = profileComponent.getConnector(mapping.getTargetConnectorId());
        if (targetConnector != null) {
            try {
                alterPrimaryKey(model, targetConnector, oldPrimaryKeys, newPrimaryKeys);
            } catch (Exception e) {
                logger.error("执行主键 DDL 失败，配置未保存", e);
                throw new RuntimeException("修改主键约束失败：" + e.getMessage(), e);
            }
        }
    }
    
    // DDL 成功后保存配置
    log(LogType.TableGroupLog.UPDATE, model);
    profileComponent.editTableGroup(model);

    // 执行自定义字段 DDL（如有）
    List<Convert> customConverts = model.getConvert();
    if (customConverts != null && !customConverts.isEmpty()) {
        // ... 自定义字段 DDL 逻辑
    }
    
    // 初始化 TableGroup
    model.isInit = false;
    model.initTableGroup(parserComponent, profileComponent, connectorFactory);
    
    List<String> list = new ArrayList<>();
    list.add(model.getId());
    submitTableGroupCountTask(mapping, list);
    return id;
}
```

```javascript
// 绑定主键顺序按钮
function bindPrimaryKeyOrderBtn() {
    $('#primaryKeyOrderBtn').on('click', function() {
        // 获取当前主键列表
        var currentPKs = $('#targetTablePK').val() || '';
        
        // 设置 tooltip
        $(this).attr('title', '当前主键顺序：' + (currentPKs || '无'));
        $('[data-toggle="tooltip"]').tooltip('fixTitle').tooltip('show');
        setTimeout(function() { $('[data-toggle="tooltip"]').tooltip('hide'); }, 2000);
        
        // 打开对话框
        showPrimaryKeyOrderModal(currentPKs);
    });
    
    // 绑定保存按钮
    $('#savePrimaryKeyOrderBtn').on('click', function() {
        // 收集新的主键顺序
        var newPKs = collectPrimaryKeyOrder();
        $('#targetTablePK').val(newPKs);
        
        // 更新字段映射表格中的主键标记
        updatePrimaryKeyMarkers(newPKs);
        
        // 关闭对话框
        $('#primaryKeyOrderModal').modal('hide');
        
        bootGrowl("主键顺序已更新!", "success");
    });
}

// 显示主键顺序对话框
function showPrimaryKeyOrderModal(currentPKs) {
    // 解析当前主键
    var pkArray = currentPKs ? currentPKs.split(',').map(s => s.trim()) : [];
    
    // 获取字段映射表格中的所有目标字段
    var targetFields = [];
    $('#fieldMappingList tr').each(function() {
        var targetField = $(this).find('td:eq(1)').text().trim();
        if (targetField) {
            targetFields.push(targetField);
        }
    });
    
    // 显示原主键顺序
    $('#oldPrimaryKeyOrder').text(currentPKs || '无');
    $('#newPrimaryKeyOrder').text(currentPKs || '无');
    
    // 生成可拖拽的主键列表
    var html = '<div class="list-group" id="primaryKeyDragList">';
    pkArray.forEach((pk, index) => {
        html += '<div class="list-group-item" style="cursor: move;" draggable="true" data-pk="' + pk + '">' +
                '<i class="fa fa-arrows"></i> ' + (index + 1) + '. ' + pk +
                '</div>';
    });
    html += '</div>';
    
    $('#primaryKeyList').html(html);
    
    // 绑定拖拽事件
    bindPrimaryKeyDragDrop();
    
    // 显示对话框
    $('#primaryKeyOrderModal').modal('show');
}

// 绑定拖拽事件
function bindPrimaryKeyDragDrop() {
    var dragSrcEl = null;
    
    $('#primaryKeyDragList .list-group-item').on('dragstart', function(e) {
        dragSrcEl = this;
        e.originalEvent.dataTransfer.effectAllowed = 'move';
        e.originalEvent.dataTransfer.setData('text/html', this.innerHTML);
        $(this).addClass('dragging');
    });
    
    $('#primaryKeyDragList .list-group-item').on('dragover', function(e) {
        if (e.preventDefault) e.preventDefault();
        e.originalEvent.dataTransfer.dropEffect = 'move';
        return false;
    });
    
    $('#primaryKeyDragList .list-group-item').on('dragenter', function(e) {
        $(this).addClass('over');
    });
    
    $('#primaryKeyDragList .list-group-item').on('dragleave', function(e) {
        $(this).removeClass('over');
    });
    
    $('#primaryKeyDragList .list-group-item').on('drop', function(e) {
        if (e.stopPropagation) e.stopPropagation();
        
        if (dragSrcEl !== this) {
            // 交换元素位置
            var srcIndex = $(dragSrcEl).index();
            var targetIndex = $(this).index();
            
            if (srcIndex < targetIndex) {
                $(this).after($(dragSrcEl));
            } else {
                $(this).before($(dragSrcEl));
            }
            
            // 更新显示顺序
            updatePrimaryKeyOrderDisplay();
        }
        
        $('.list-group-item').removeClass('dragging over');
        return false;
    });
    
    $('#primaryKeyDragList .list-group-item').on('dragend', function(e) {
        $(this).removeClass('dragging');
        $('.list-group-item').removeClass('over');
    });
}

// 更新主键顺序显示
function updatePrimaryKeyOrderDisplay() {
    var newPKs = collectPrimaryKeyOrder();
    $('#newPrimaryKeyOrder').text(newPKs || '无');
}

// 收集新的主键顺序
function collectPrimaryKeyOrder() {
    var pks = [];
    $('#primaryKeyDragList .list-group-item').each(function() {
        pks.push($(this).data('pk'));
    });
    return pks.join(',');
}

// 更新字段映射表格中的主键标记
function updatePrimaryKeyMarkers(pkString) {
    var pkArray = pkString ? pkString.split(',').map(s => s.trim()) : [];
    
    $('#fieldMappingList tr').each(function() {
        var targetField = $(this).find('td:eq(1)').text().trim();
        var $pkCell = $(this).find('td:eq(2)');
        
        if (pkArray.includes(targetField)) {
            if ($pkCell.find('i').length === 0) {
                $pkCell.html('<i title="主键" class="fa fa-key fa-fw text-warning"></i>');
            }
        } else {
            $pkCell.html('');
        }
    });
}
```

## 方案对比

### 原方案 vs 修正方案

| 方面 | 原方案 | 修正方案 |
|------|--------|----------|
| **持久化** | 无（targetTablePK 未保存） | ✅ TableGroup 新增 targetTablePK 字段 |
| **DDL 执行** | 直接 executeDDL() | ✅ connectorFactory.writerDDL() 框架 |
| **SQL 前缀** | 无 | ✅ /*dbs*/ 防止双向同步循环 |
| **数据库支持** | 仅 MySQL | ✅ MySQL + SQL Server |
| **事务顺序** | 先保存后 DDL | ✅ 先 DDL 后保存 |
| **前端 UI** | 无 | ✅ 主键顺序按钮 + 对话框 |
| **风险提示** | 无 | ✅ 红色警告框 + 注意事项 |

---

## 影响范围

### 正面影响
- ✅ 用户可以修正主键配置错误，无需删除重建
- ✅ 提升用户体验（主键顺序可视化调整）
- ✅ 降低误操作风险（明确的风险提示）
- ✅ 支持 MySQL 和 SQL Server 两种主流数据库

### 潜在风险
- ⚠️ `TableGroup` 新增字段需要兼容旧数据（反序列化时 targetTablePK 为 null）
- ⚠️ SQL Server 主键约束名查询可能需要优化（当前简化实现为 `PK_表名`）
- ⚠️ DDL 执行失败后，配置未保存（需要用户重新操作）

### 兼容性处理
```java

}
```
